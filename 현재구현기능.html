1. 프로젝트 요약

나무에 마커를 부착하고, 스마트폰 브라우저에서 카메라로 마커를 인식하여 나무의 높이·폭을 측정하는 WebAR 앱이다. 설치 없이 URL 접근만으로 동작한다.

✅
현재 상태: Phase 1 프로토타입 완성 — 카메라 연동, 기본 마커 검출, 터치 2점 높이 계산, 결과 저장 기능이 구현되어 있다.

2. 파일 구조

/tree-ar-app ├── index.html ← 랜딩 페이지 (측정 시작 CTA) ├── measure.html ← AR 측정 화면 (카메라 + 오버레이) ├── result.html ← 결과 화면 (수치 카드 + 저장/공유) ├── css/ │ └── style.css ← 디자인 시스템 (다크 테마 + 글래스모피즘) ├── js/ │ ├── app.js ← 메인 컨트롤러 (RAF 루프 + 상태 관리) │ ├── camera.js ← 카메라 모듈 (getUserMedia + 토치) │ ├── detector.js ← 마커 검출 (Phase 1 기본 검출기) │ ├── measure.js ← 측정 계산 (픽셀→실제 높이 변환) │ └── storage.js ← IndexedDB 데이터 저장 ├── markers/ ← 마커 패턴 (Phase 2에서 추가 예정) └── assets/ ← 아이콘, 로고

4. 화면 플로우 (구현 완료)

① 랜딩 (index.html)

🌳 아이콘 플로팅 애니메이션 + "측정 시작" 버튼 + 기능 카드 3개

② 카메라 권한 요청 (measure.html)

로딩 스피너 → getUserMedia 호출 → 허용 시 AR 화면 전환, 거부 시 에러 + 재시도

③ 마커 스캐닝

풀스크린 카메라 피드 + "마커 검색 중..." 노란색 상태 배지 + 크로스헤어

④ 마커 인식 완료

"마커 인식됨" 녹색 배지 + 마커 윤곽 오버레이 + 거리 텍스트 + 캡처 버튼 활성화

⑤ 터치 포인트 입력 (측정)

꼭대기 터치(첫 번째) → 밑동 터치(두 번째) → 점선 라인 + 실시간 높이 계산·표시

⑥ 결과 페이지 (result.html)

캡처 이미지 + 높이/폭/거리 데이터 카드 + 💾 저장(IndexedDB) + 📤 공유(Web Share)

5. 모듈별 주요 API

Camera (camera.js)

메서드 파라미터 반환 설명
Camera.init(video) HTMLVideoElement Promise<MediaStream> 후면 카메라 스트림 연결
Camera.toggleTorch() — Promise<boolean> 플래시 토글, 현재 상태 반환
Camera.getResolution() — {width, height} 비디오 해상도
Camera.captureFrame() — string (data URL) 현재 프레임 JPEG 캡처
Camera.stop() — void 스트림 해제

Detector (detector.js)

메서드 파라미터 반환 설명
Detector.init(options) {markerSize, onFound, onLost} void 검출기 초기화, 마커 크기(m) 설정
Detector.detect(imageData) ImageData Object | null 프레임에서 마커 검출. {id, corners, confidence, pixelSize}
Detector.estimateDistance(px) pixelSize, [focalLength] number d = (f × S) / p 거리 추정(m)

Measure (measure.js)

메서드 파라미터 반환 설명
Measure.init(opts) {videoWidth, videoHeight, displayWidth, displayHeight} void 비디오/화면 좌표계 초기화
Measure.addPoint(x, y) screenX, screenY {index, point} 터치 포인트 추가 (최대 2점, 초과 시 리셋)
Measure.calculateHeight(d) distance, [focalLength] {height, width, distance} 픽셀 차이 기반 실제 높이 계산
Measure.reset() — void 포인트 초기화

TreeStorage (storage.js)

메서드 파라미터 반환 설명
TreeStorage.save(data) {height, width, distance, imageData, …} Promise<number> IndexedDB에 저장 + GPS 자동 취득
TreeStorage.getAll() — Promise<Array> 전체 측정 기록 조회 (최신순)
TreeStorage.remove(id) id Promise 특정 기록 삭제

6. 측정 원리 (현재 구현)

6-1. 거리 추정

d = (f × S) / p

d: 카메라-마커 거리 (m)
f: 카메라 초점거리 (px) — 기본값 800
S: 마커 실제 크기 (m) — 기본값 0.20 (20cm)
p: 마커의 이미지 상 픽셀 크기

6-2. 높이 계산

h = d × (Δpx / f)

h: 실제 높이 (m)
d: 거리 (m) — 위 공식으로 추정
Δpx: 꼭대기·밑동 두 터치점의 y축 픽셀 차이
f: 카메라 초점거리 (px)

6-3. RAF 루프 파이프라인

requestAnimationFrame(tick)
  │
  ├─ 1. ctx.drawImage(video, 0, 0, 320, 240)     축소 프레임 캡처
  │
  ├─ 2. Detector.detect(imageData)                마커 검출
  │     ├─ 검출 성공 → onFound 콜백 + 상태 전환
  │     └─ 15프레임 연속 미검출 → onLost 콜백
  │
  ├─ 3. drawOverlay(marker)                       캔버스 오버레이 렌더링
  │     ├─ 마커 윤곽 (녹색 사각형)
  │     ├─ 중심 십자 + 거리 텍스트
  │     ├─ 터치 포인트 (원형 마커)
  │     └─ 측정 라인 (점선 + 높이 라벨)
  │
  └─ 4. requestAnimationFrame(tick)               다음 프레임

10. 실행 방법

로컬 테스트 (데스크톱)

# 프로젝트 디렉토리에서 HTTP 서버 실행
cd /Users/lucas/dev/tree_measurements/tree-ar-app
python3 -m http.server 8080

# 브라우저에서 접속
# http://localhost:8080

모바일 테스트 (실제 카메라 사용)

# HTTPS 필요 (카메라 권한)
# 방법 1: ngrok 사용
ngrok http 8080

# 방법 2: GitHub Pages 배포
# Settings → Pages → Source: main → /tree-ar-app
⚠️
주의: 카메라 API는 HTTPS 또는 localhost에서만 동작합니다. 모바일 실기기 테스트 시 반드시 HTTPS로 서빙하세요.
