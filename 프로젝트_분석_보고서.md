# TreeMeasure 프로젝트 분석 보고서

작성일: 2026-03-29

---

## 1. 프로젝트 개요

**TreeMeasure**는 스마트폰 브라우저에서 실행되는 WebAR 기반 나무 측정 애플리케이션이다. AR 마커를 활용해 나무의 **수고(높이)**와 **수관폭(너비)**을 현장에서 즉시 측정할 수 있도록 설계되었으며, GPS 좌표와 함께 측정 결과를 저장·공유하는 기능을 제공한다.

### 적용 분야
- 산림 조사 및 수목 관리
- 환경 모니터링
- 조경 설계 지원

## 3. 기술 스택

### 프론트엔드
- **HTML5 / CSS3 / Vanilla JavaScript (ES6+)**
- **A-Frame 1.5.0** — WebGL/WebXR 3D 렌더링 프레임워크
- **AR.js** — 마커 기반 증강현실 감지
- **js-aruco2** — ArUco 마커 대체 감지 라이브러리

### 브라우저 API
| API | 사용 목적 |
|-----|-----------|
| `getUserMedia()` | 카메라 접근 |
| `Geolocation API` | GPS 좌표 수집 |
| `IndexedDB` | 측정 이력 로컬 저장 |
| `Canvas API` | AR 오버레이 그리기 |
| `MediaStream API` | 손전등(Torch) 제어 |
| `Web Share API` | 결과 네이티브 공유 |

## 4. 구현 기능 분석

### 4.1 마커 감지 (detector.js)

2단계 폴백(Fallback) 구조를 통해 안정성을 확보했다.

```
1차: AR.js (Hiro + 커스텀 패턴 마커) → 3D 포즈 추정 제공
2차: js-aruco (ArUco 바이너리 마커) → AR.js 실패 시 픽셀 기반 추정
```

- 마커 손실 시 500ms 디바운싱으로 상태 전환 안정화
- THREE.js 3D 벡터로 카메라-마커 간 거리 계산

### 4.2 측정 엔진 (measure.js)

**측정 공식:**

```
거리(m) = 카메라-마커 거리(d) × (픽셀 차이(Δpx) / 초점 거리(f))
```

- `d`: AR 마커로부터 추정한 카메라 거리 (미터)
- `Δpx`: 화면상 두 터치 포인트 간 픽셀 차이
- `f`: 초점 거리 (고정값 800~1000px — 기기별 미보정)

**측정 모드:**
- **수고 모드**: 나무 꼭대기 → 밑동 터치 (수직 거리)
- **수관폭 모드**: 좌측 끝 → 우측 끝 터치 (수평 거리)

### 4.3 상태 머신 (app.js)

```
LOADING → SCANNING → READY → MEASURING → DONE → ERROR
```

각 상태에서 유효하지 않은 동작을 방지하며, 마커 감지 상태에 따라 UI가 반응형으로 변화한다.

### 4.4 데이터 저장 (storage.js)

- **IndexedDB v2** 스키마: 측정 이력 영구 저장
- 인덱스: `timestamp`, `latitude`, `longitude`
- 세션 간 이력 조회 및 삭제 지원
- `sessionStorage` 활용: 측정 → 결과 페이지 간 임시 데이터 전달

### 4.5 카메라 제어 (camera.js)

- 후면 카메라 우선 선택 (`facingMode: environment`)
- 목표 해상도: 1280×720
- 손전등(Torch) 지원 감지 및 토글
- JPEG 프레임 캡처 기능

## 5. 코드 품질 평가

### 강점

**1. 모듈화 설계**
각 모듈이 단일 책임 원칙을 따르며 IIFE 패턴으로 캡슐화되어 있다.

```javascript
const Measure = (() => {
    let points = [];  // 비공개 상태
    return { init, calculate, reset };  // 공개 API
})();
```

**2. 폴백 아키텍처**
AR.js 실패 시 js-aruco로 자동 전환하는 이중화 구조로 현장 신뢰성을 높였다.

**3. 에러 처리**
- AR 엔진 로딩 타임아웃 (20초)
- 로딩 실패 시 재시도 버튼 제공
- 카메라 권한 거부 시 안내 메시지

**4. 모바일 최적화**
- 터치 이벤트 핸들링
- `user-scalable=no` viewport 설정
- 손전등 지원 기기 감지
- requestAnimationFrame 기반 렌더링 루프

### 약점 및 개선 필요 사항

| 구분 | 내용 | 심각도 |
|------|------|--------|
| **정확도** | 초점 거리(f)가 800px 고정값 — 기기별 미보정 | 중 |
| **오프라인** | A-Frame·AR.js를 CDN에서 로드 — 인터넷 없으면 동작 불가 | 중 |
| **메모리 누수** | `watchPosition()` 해제 로직 없음 | 중 |
| **테스트** | 자동화 테스트(단위/통합) 없음 | 중 |
| **세션 손실** | 측정 중 페이지 새로고침 시 데이터 유실, 미안내 | 낮음 |
| **TypeScript** | 타입 안전성 없음 | 낮음 |

## 6. 잠재적 버그

1. **측정 모드 전환 시 포인트 초기화**: 배열은 지워지나 UI 반영이 지연될 수 있음
2. **거리 검증**: 비정상 거리 값을 5m로 클램핑하나 사용자에게 미안내
3. **3번째 터치 포인트**: 2점 초과 입력 시 UI 피드백 없음
4. **캔버스 리사이즈**: measure.js와 표시 크기 동기화 깨질 가능성

---

## 7. 측정 정확도

문서에 명시된 예상 정확도는 **±5~15%** 수준이다.

**정확도 제한 원인:**
- 초점 거리가 기기별로 보정되지 않음 (고정값 사용)
- 렌즈 왜곡 보정 없음
- 지면이 평탄하다고 가정 (경사지 미보정)
- GPS 정확도 ±10~100m (환경에 따라 변동)
- 사용자 손떨림 및 시차 오류

## 9. 종합 평가

TreeMeasure는 **명확한 아키텍처와 모바일 최적화가 잘 된 WebAR 측정 애플리케이션**이다.
이중 마커 감지 시스템, GPS 연동, IndexedDB 저장 등 현장 활용에 필요한 핵심 기능이 구현되어 있으며, 한국어 문서화도 충실하게 작성되어 있다.

**즉시 개선 권장 사항 (우선순위 순):**

1. `watchPosition()` 언마운트 시 정리 코드 추가 (메모리 누수 방지)
2. A-Frame/AR.js 로컬 번들링 또는 서비스 워커 캐싱 (오프라인 지원)
3. 기기별 초점 거리 보정 로직 구현 (측정 정확도 향상)
4. 핵심 모듈 단위 테스트 추가
5. 측정 중 새로고침 경고 다이얼로그 추가

Phase 3의 WebXR Depth Sensing이 구현되면 측정 정확도를 크게 향상시킬 수 있을 것으로 기대된다.
