1. 프로젝트 개요

목적

나무에 마커를 부착하고, 스마트폰 브라우저에서 카메라로 마커를 인식하여 나무의 높이·폭·흉고직경을 측정하는 WebAR 앱

핵심 가치

  • 설치 불필요 — URL 접근만으로 즉시 사용
  • 마커 기반 — 거리 자동 계산, 사용자 입력 최소화
  • 하이브리드 검출 — AR.js + js-aruco 병행으로 현장 신뢰성 확보

목표 정확도

조건 예상 오차
20cm 마커, 5m 거리 ±5–8%
15cm 마커, 3m 거리 ±3–5%
역광/조명 불량 ±10–15%

2. 측정 방식 결정

가이드 문서에서 분석된 3가지 방식 중 마커 기반 AR 측정을 1차 방식으로 채택한다.

방식 장점 단점 채택
삼각측량 + 센서 WebXR 불필요, 단순 구현 사용자 거리 입력 필요, ±5-15% 오차 폴백
WebXR Depth Sensing 자동화 유리 브라우저 지원 제한적 v3 확장
마커 기반 AR 거리 자동 계산, ±2-5% 오차 마커 부착 필요 ✅ 메인

측정 원리

거리 추정:  d ≈ (f × S) / p
높이 계산:  h = d × tan(θ_top) - d × tan(θ_bottom)

f: 카메라 초점거리(px), S: 마커 실제 크기(m), p: 마커 이미지 픽셀 크기

3. 기술 스택

역할 기술 선택 이유
마커 검출 (1차) AR.js 60fps 모바일 최적화, Hiro/Custom .patt 지원
마커 검출 (폴백) js-aruco2 ArUco ID 오류 정정, 노이즈 환경 강인
3D 렌더링 Three.js 측정 라인/텍스트 오버레이
AR 씬 관리 A-Frame AR.js 연동, 컴포넌트 기반 선언적 구조
카메라 접근 getUserMedia facingMode: environment (후면 카메라)
위치 기록 Geolocation API 측정 지점 GPS 좌표 저장
로컬 저장 IndexedDB 오프라인 환경 대응
빌드/개발 Vite 빠른 HMR, ES modules 지원

하이브리드 검출 전략

프레임 입력 → AR.js detect (2–5ms)
                ↓ 실패?
              js-aruco detect (5–10ms)
                ↓ 실패?
              "마커 없음" (이전 프레임 스무딩)

병행 효과: 단일 라이브러리 대비 검출 성공률 20–30%↑

4. 마커 설계

크기 기준

나무 크기 마커 촬영 거리 화면 픽셀 검출률
소형 (≤5m) 15cm 2–3m 80–150px 95%+
중형 (5–15m) 20cm 4–6m 60–120px 90–95%
대형 (15m+) 25cm 6–10m 50–100px 85–90%
ℹ️
권장 크기: 20cm — A4 용지 한 변 크기로 출력 용이, 5–8m 거리에서 안정적 검출

마커 제작

  • 생성: AR.js marker-training generator에서 300dpi PDF
  • 재질: 방수 라미네이트 스티커, 검정 테두리 필수
  • 부착: 트렁크 흉고(1.3m) 위치, 정면 향하게 고정
  • 전략: 프로토타입 → Hiro 표준 마커 / 배포 → Custom .patt (검출률 +15%)

8. 핵심 구현 사항

8-1. RAF 루프 (핵심 파이프라인)

JavaScript 
const MARKER_SIZE = 0.20;  // 20cm

function tick() {
  // 1. 프레임 캡처
  ctx.drawImage(video, 0, 0, W, H);
  const imageData = ctx.getImageData(0, 0, W, H);

  // 2. 병행 검출
  let markers = arJsDetector.detect(imageData);
  if (!markers.length) markers = jsArucoDetector.detect(imageData);

  // 3. 포즈 + 측정
  if (markers.length) {
    const matrix = estimatePose(markers[0].corners, MARKER_SIZE);
    threeMarker.matrix.copy(matrix);
    if (touchPoints.length === 2) {
      const height = computeHeight(touchPoints, matrix);
      heightLabel.setText(`높이: ${height.toFixed(2)}m`);
    }
    showMarkerFound();
  } else {
    showMarkerLost();
  }

  // 4. 렌더
  renderer.render(scene, camera);
  requestAnimationFrame(tick);
}

9. 개발 로드맵

Phase 1 프로토타입 1–2주
  • 프로젝트 초기 셋업 (Vite + 기본 파일 구조)
  • 랜딩 페이지 UI (측정 시작 버튼)
  • getUserMedia 카메라 연동
  • Hiro 마커 + AR.js + A-Frame 기본 셋업
  • 마커 인식 → 3D 축 오버레이 표시
  • 터치 2점 입력 → 높이 계산 및 출력
Phase 2 기능 강화 3–4주
  • js-aruco 폴백 로직 추가 (하이브리드 검출)
  • Custom .patt 마커 생성 및 교체
  • 폭(수관폭) 측정 기능 추가
  • GPS 위치 + 사진 캡처 저장 (IndexedDB)
  • 결과 화면 UI 구현 (수치 카드, 공유)
  • 마커 스무딩 (이전 N프레임 평균)
Phase 3 배포 및 최적화 5–6주
  • Web Workers로 검출 오프로드 (FPS 안정화)
  • 저조도 환경 토치 버튼
  • 서버 업로드 API 연동
  • HTTPS 배포 (카메라/GPS 권한 필수)
  • 모바일 Chrome/Safari 크로스 브라우저 테스트
  • 성능 최적화 (저사양: 320×240 검출 + 업스케일)
Phase 4 고급 기능 (향후) 추후
  • WebXR Depth Sensing 지원 디바이스 자동 모드
  • 삼각측량 센서 모드 폴백 (마커 없이 측정)
  • 다중 나무 배치 측정 (ArUco ID 활용)

10. 검증 계획

자동화 테스트

  • Vite 빌드 검증: npm run build — 에러 없이 번들링 완료 확인
  • Lint/Type 검사: ESLint 설정 및 실행

브라우저 테스트

  • browser_subagent 도구로 랜딩 페이지 로드 → 버튼 렌더링 확인
  • 카메라 권한 플로우 UI 전환 확인 (카메라 없는 환경에서는 에러 UI 확인)

수동 검증 (실제 기기 필요)

  1. 마커 인식 테스트: 20cm Hiro 마커 출력 → 3m/5m/8m 거리에서 인식률 확인 (목표: 90%+)
  2. 측정 정확도 테스트: 줄자로 실측한 높이와 앱 측정값 비교 (목표: ±10% 이내)
  3. 크로스 브라우저: iOS Safari / Android Chrome에서 카메라 → 마커 인식 → 측정 전체 플로우
⚠️
AR 기능은 실제 카메라와 마커가 필요하므로, 실기기 수동 테스트가 필수적입니다.

11. 배포 체크리스트

환경

  • HTTPS 필수 (카메라/GPS 권한)
  • 모바일 Chrome/Safari 테스트 필수
  • 후면 카메라 (facingMode: environment)

현장 준비

  • 20cm Hiro 마커 방수 출력
  • 나무 트렁크 1.3m 위치 부착
  • 5m, 8m 거리 검출률 90%+ 확인

성능

  • 640×480 해상도 (성능 균형)
  • requestAnimationFrame 렌더 루프
  • 저사양 기기: 320×240 검출 + 업스케일
